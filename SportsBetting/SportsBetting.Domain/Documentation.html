Sports Betting Domain Model

Complete API Documentation - Pre-Game Betting Platform

Version 1.0 | .NET 9.0 | Pure OOP Domain Model

Overview

This domain model provides a complete object-oriented implementation of a sports betting platform. It supports both traditional bookmaker and peer-to-peer exchange models with pre-game betting capabilities.

Architecture Highlights

  • Pure Domain Logic: No database, framework, or infrastructure dependencies
  • Rich Domain Model: Entities contain business logic, not just data
  • Type Safety: Value objects prevent invalid states (negative money, invalid odds)
  • Immutability: Locked odds, immutable value objects for thread safety
  • Settlement Engine: Complete bet grading with void/push handling

Supported Features

Market Types
  • Moneyline
  • Point Spread
  • Totals (Over/Under)
  • Props & Futures (structure ready)
Bet Types
  • Single Bets
  • Parlays (2+ legs)
  • System Bets (structure ready)
  • Round Robin (structure ready)
Settlement
  • Automatic outcome grading
  • Void leg handling
  • Push scenarios
  • Event cancellation

Settlement Logic

Single Bet:
  • Win → Pay full payout
  • Loss → Stake lost
  • Push/Void → Return stake

Parlay:
  • All win → Pay full payout
  • Any loss → Bet loses
  • Void leg(s) → Recalculate odds without void legs
  • All void → Return stake

Services

Domain services that orchestrate business logic across multiple entities. Services contain operations that don't naturally belong to a single entity.

SettlementService

public class SettlementService
Service for settling bets based on event outcomes. Handles different market types (Moneyline, Spread, Totals) and orchestrates the settlement of individual selections and complete bets.

Methods

public void SettleMoneylineMarket(Market market, Event evt)
Settles a moneyline market based on final score. Determines winner based on which team scored more points.
Parameters:
market: Market - The moneyline market (must be MarketType.Moneyline)
evt: Event - The event with final score
Behavior:
  • If draw and no draw outcome exists: voids both outcomes (push)
  • Otherwise marks the winning team's outcome
public void SettleSpreadMarket(Market market, Event evt)
Settles a point spread market based on final score. Applies the spread to determine which outcome covers.
Parameters:
market: Market - The spread market (must be MarketType.Spread)
evt: Event - The event with final score
Behavior:
  • Applies line to home score: adjustedScore = homeScore + line
  • If adjusted score = away score: Push (void both outcomes)
  • Otherwise marks the covering outcome
public void SettleTotalsMarket(Market market, Event evt)
Settles a totals (over/under) market based on final score. Compares combined score to the line.
Parameters:
market: Market - The totals market (must be MarketType.Totals)
evt: Event - The event with final score
Behavior:
  • Calculates total points = homeScore + awayScore
  • If total = line: Push (void both outcomes)
  • If total > line: Over wins
  • If total < line: Under wins
public void SettleEvent(Event evt)
Settles all markets for an event. Automatically determines market type and applies appropriate settlement logic.
Parameters:
evt: Event - The completed event
public void SettleBet(Bet bet, IEnumerable<Event> events)
Settles a bet based on its selections and current market states. First settles all individual selections, then settles the bet with appropriate parlay logic (void handling, recalculation).
Parameters:
bet: Bet - The bet to settle
events: IEnumerable<Event> - Collection of events referenced by bet selections
